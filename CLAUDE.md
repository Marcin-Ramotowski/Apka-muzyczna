# Context File — Apka-muzyczna

## Code Style

1. **Indentation**: 4 spaces, no tabs.
2. **Naming**: Variables/functions: `snake_case`; Classes: `PascalCase`; Constants: `UPPER_SNAKE_CASE`. Follow Django conventions.
3. **Line Length**: Max ~100 characters.
4. **Comments**: Concise and meaningful. Explain sections or non-obvious logic.
5. **Django Style**: Follow "Django way". Standard structure: `models.py`, `views.py`, `urls.py`, `forms.py`. Avoid custom abstractions.
6. **Testing**: Use `pytest` in `music/tests.py`. Focus on core functionality.

## Workflow

- **Start with Docker Compose**: `docker compose up --build`
  - migrations are applied automatically before Django starts
- **Stop Compose**: `docker compose down`
- **Run tests**: `docker compose run --rm web pytest`
- **Alternative Docker commands**:
  - `docker build -t apka-muzyczna .`
  - `docker run -p 8000:8000 apka-muzyczna`
- **Optional local development**:
  - `python -m venv venv`
  - `source venv/bin/activate`
  - `pip install -r requirements.txt`
  - `python manage.py runserver`

## Constraints

- No raw SQL; use Django ORM.
- Logic in `views.py`; no business logic in templates.
- Follow Django patterns; prioritize readability.
- No hardcoded secrets; use .env.
- Minimal dependencies.
